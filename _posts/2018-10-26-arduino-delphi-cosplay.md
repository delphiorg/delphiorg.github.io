<p>Comic conventions are the usual place to find cosplay, but Halloween is also a good excuse for some cosplay fun. I'm a huge fan of creative cosplay and love getting my picture with cosplayers. </p>

<p>This year I am putting together my <strong>Ultimate Cyberpunk cosplay</strong>. The first part is a jacket based on the main character from the game <em><a href="https://www.cyberpunk.net/en/">Cyberpunk 2077</a></em><a href="https://www.cyberpunk.net/en/"> by CD Projekt RED</a>. <em><a href="https://en.wikipedia.org/wiki/Neuromancer">Neuromancer</a></em><a href="https://en.wikipedia.org/wiki/Neuromancer">, by William Gibson</a>, inspired the <em><a href="https://en.wikipedia.org/wiki/Cyberpunk_2020">Cyberpunk 2020</a></em><a href="https://en.wikipedia.org/wiki/Cyberpunk_2020"> RPG by Mike Pondsmith</a>. They both inspired this unreleased video game <em>Cyberpunk 2077</em>. </p>

<p>I'm a big fan of both the book and the original RPG and am looking forward to the video game. I combined elements from all three sources for my cosplay. The character V is more of a Street Samurai / Solo, while I wanted to go for more of a Decker / Netrunner (a little <a href="https://en.wikipedia.org/wiki/Shadowrun">Shadowrun RPG</a> influence in there too).</p>

<figure class="wp-block-pullquote is-style-solid-color"><blockquote><p>At this point the secret is out, I'm a huge nerd. I love Role Playing Games, and Cyberpunk &amp; Shadowrun are my favorite genres.</p></blockquote></figure>

<p>I didn't have my RGB Shades in time for <a href="https://fanxsaltlake.com/">Salt Lake FanX</a>, so I justed glued some extra LEDs (NeoPixel 4x4 grids) to my face. They represent cybernetic enhancements, which is why I left the wires exposed. I used an Arduino MEGA 2560 by ELEGOO as the controller. It also controlled the string of NeoPixel lights in the collar. I attached a Bluetooth keyboard to one wrist and an Android Pixel phone to the other. The phone was decorative&nbsp;<em>for now</em>.</p>

<p>Later I got my <a href="http://macetech.com/store/index.php?main_page=product_info&amp;cPath=14&amp;products_id=59">RGB LED Shades</a> which will become the key to my outfit for this Halloween. <p>After soldering the HC-06 onto my shades I can power them up and pair my Android phone with the HC-06 module. One note about my soldering is that if you look you will see that it is connected to two analog pins instead of digital pins. On most Arduinos the analog pins can double as digital pins (see the <a href="/assets/images/2018/10/mini-1.png">Pinout</a>).</p>

<figure class="wp-block-image"><img src="https://i0.wp.com/delphi.org/assets/images/2018/10/BluetoothDialog.jpg?fit=636%2C1024" alt="Android Bluetooth Pairing Dialog" class="wp-image-3190"/><figcaption>Android Bluetooth Pairing Dialog</figcaption></figure>

<p>After that, a <a href="http://docwiki.embarcadero.com/RADStudio/en/Using_Classic_Bluetooth">TBluetooth</a> component is able to open a socket to the shades. Start the pairing with <code>Bluetooth1.<a href="http://docwiki.embarcadero.com/Libraries/en/System.Bluetooth.Components.TBluetooth.DiscoverDevices">DiscoverDevices</a>( 5000 );</code> In the <a href="http://docwiki.embarcadero.com/Libraries/en/System.Bluetooth.Components.TBluetooth.OnDiscoveryEnd">DiscoveryEnd</a> event handler the following code will open a socket to the HC-06 module (Thanks to <a href="http://mitov.com/">Boian Mitov</a> for the basis of this code):</p>

<pre>[delphi]var
  ADevice   : TBluetoothDevice;
  AService  : TBluetoothService;
begin
  for ADevice in ADeviceList do
  begin
    // HC-06 is the name of the bluetooth device
    if ADevice.DeviceName = 'HC-06' then 
    begin
      Bluetooth1.Pair( ADevice );
      for AService in ADevice.LastServiceList do
      begin
        // FSocket is a TBluetoothSocket with larger scope
        FSocket := ADevice.CreateClientSocket( 
            AService.UUID, False );
        If Assigned( FSocket ) then
        begin
          FSocket.Connect;
          Break;
        end;
      end;
      Break;
    end;
  end;
end;[/delphi]</pre>

<p>Once the <a href="http://docwiki.embarcadero.com/Libraries/en/System.Bluetooth.TBluetoothSocket">socket</a> is open you can send characters with this code.</p>

<pre class="wp-block-preformatted">[delphi]FSocket.SendData( TEncoding.UTF8.GetBytes( 'D' ));[/delphi]</pre>

<p>It is so easy to work with <a href="http://docwiki.embarcadero.com/RADStudio/en/Using_Classic_Bluetooth">Classic Serial Bluetooth</a> in Delphi (or C++Builder) and the <a href="http://docwiki.embarcadero.com/Libraries/en/System.Bluetooth.Components.TBluetooth">TBluetooth component</a>. You even could send a whole string or other binary data. For this project a single character was all I needed to change modes on the shades. A single character is also easier to process on the Arduino side. (Technically <a href="http://docwiki.embarcadero.com/RADStudio/en/Using_Classic_Bluetooth">Bluetooth Classic</a> works on all platforms, but on iOS you need special <a href="https://developer.apple.com/programs/mfi/">approval from Apple on a per app basis</a>.)</p>

<p>On the Arduino side, you can use the <a href="https://www.arduino.cc/en/Main/Software">Arduino IDE</a> and flex your C programming skills. Or you can do like I usually do and use <a href="https://www.visuino.com/">Visuino</a> by Boian Mitov of <a href="http://mitov.com">Mitov Software</a>. It provides a visual drag and drop interface for programming Arduino devices. It won the <a href="https://www.embarcadero.com/press-releases/embarcadero-announces-april-cool-app-visuino">Embarcadero Cool App contest in April 2017</a> as Boian used Delphi to develop Visuino. Boian also recently added <a href="https://labpacks.blogspot.com/search?q=%23Macetech">RGB LED Shades</a> support to Visuino (along with unboxing and assembly videos.)</p>
